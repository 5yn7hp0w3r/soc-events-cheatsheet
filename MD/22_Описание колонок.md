Помогает быстро фильтровать ошибки по типу проблемы. |

|                                              | Принципы шкалы:                                                                                                                                                                                                                                                                   |              |                                           |                                                                                                                                                                                   |
|                                              | • «Критическая» резервируется за событиями, которые почти всегда означают компрометацию или сокрытие следов (очистка/остановка журналов, отключение защиты, WMI-persistence, доступ к хешам).                                                                                     |              |                                           |                                                                                                                                                                                   |
|                                              | • Шумные высокочастотные события (4624, 4768/4769, 5140) базово «Средние/Низкие» и повышаются только в корреляции (см. колонку «Корреляции»).                                                                                                                                     |              |                                           |                                                                                                                                                                                   |
|                                              | • Целевое распределение — «пирамида» (мало критических, основная масса в Средней/Низкой), а не преобладание «Высоких» — иначе возникает alert fatigue.                                                                                                                            |              |                                           |                                                                                                                                                                                   |
| Частота                                      | Высокая / Средняя / Низкая / Очень низкая                                                                                                                                                                                                                                         |              | Значение для SOC                          | Интерпретация для аналитика: что может означать появление этой ошибки с точки зрения кибербезопасности — от нормальной операции до индикатора атаки.                              |
|                                              | Оценка того, как часто это событие встречается в нормальных условиях эксплуатации. Высокая частота может означать «шумность» события.                                                                                                                                             |              | Рекомендации                              | Практические советы по мониторингу и реагированию: на что обращать внимание и как использовать этот код ошибки для обнаружения угроз.                                             |
| Описание                                     | Краткое, емкое объяснение сути события: что произошло и в каком контексте.                                                                                                                                                                                                        |              |                                           |                                                                                                                                                                                   |
| Подробности                                  | Детализация о том, какую конкретную информацию содержит запись события в логе.                                                                                                                                                                                                    |              |                                           |                                                                                                                                                                                   |
| Индикаторы атаки (IoC)                       | Конкретные признаки в контексте этого события, которые с высокой вероятностью указывают на злонамеренную активность (например, имена специфических процессов, параметры, временные метки).                                                                                        |              |                                           |                                                                                                                                                                                   |
| Рекомендации по мониторингу (планируется)    | Практические советы по тому, на что обращать внимание и как эффективно выявлять аномалии, связанные с этим событием.                                                                                                                                                              |              |                                           |                                                                                                                                                                                   |
| Пример SIEM-запроса (планируется)            | Готовый шаблон запроса (часто для Splunk или аналогичных систем) для поиска подозрительных экземпляров данного события.                                                                                                                                                           |              |                                           |                                                                                                                                                                                   |
| MITRE TTP                                    | Идентификатор тактики, техники или процедуры (TTP) по фреймворку MITRE ATT&CK, с которой связано данное событие. Позволяет понять мотив и метод злоумышленника.                                                                                                                   |              |                                           |                                                                                                                                                                                   |
| Причины отказа                               | Объяснение, почему операция, связанная с событием, могла не произойти или завершиться ошибкой (например, недостаток привилегий, блокировка политикой).                                                                                                                            |              |                                           |                                                                                                                                                                                   |
| Типичные сценарии                            | Примеры легитимной, нормальной активности, которая также вызывает данное событие. Необходимо для снижения ложных срабатываний.                                                                                                                                                    |              |                                           |                                                                                                                                                                                   |
| Ключевые поля                                | Названия наиболее важных полей данных в записи события, которые необходимо анализировать для расследования (например, ProcessName, SourceIP).                                                                                                                                     |              |                                           |                                                                                                                                                                                   |
| Причины блокировки                           | Факторы (политики, правила), которые могут предотвратить саму возможность возникновения этого события (например, правило брандмауэра блокирует подключение).                                                                                                                      |              |                                           |                                                                                                                                                                                   |
| Ключевые привилегии                          | Специфические права Windows (например, SeDebugPrivilege), требующиеся учетной записи или процессу для выполнения операции, ведущей к событию.                                                                                                                                     |              |                                           |                                                                                                                                                                                   |
| Документация                                 | Ссылка на официальный источник информации (обычно docs.microsoft.com) для данного Event ID.                                                                                                                                                                                       |              |                                           |                                                                                                                                                                                   |
