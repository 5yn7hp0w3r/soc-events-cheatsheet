📖Как пользоваться поиском

Короткая инструкция по интерактивному справочнику. Всё работает в браузере, без установки.

🚀 Быстрый старт

  1. Открой 🔎 Поиск событий.
  2. Вбей номер события — например 4625 — или ключевое слово (mimikatz, lsass).
  3. Кликни по карточке, чтобы раскрыть детали: индикаторы атаки, False Positive, SPL-запрос, корреляции.
💡 Нажми / в любой момент — курсор прыгнет в поиск. Esc — очистить.

🔢 Поиск по нескольким Event ID сразу

Вставь несколько номеров через запятую или пробел — покажет ровно эти события. Удобно, когда в SIEM всплыла пачка кодов.

4624, 4688, 4625  или  4624 4688 4625
Числовой запрос автоматически ищет точно по Event ID — без «шума» из корреляций и SPL других событий.

📊 Счётчики событий

Укажи в скобках, сколько раз событие встретилось в твоих логах — справочник покажет это на карточке и отсортирует по частоте (самые частые сверху).

4624(25) 4625(1337) 4672(48)
→ на карточке 4625 появится 📊 Количество событий: 1337
Так сразу видно, что 1337 неудачных входов (4625) — это аномалия, на которую стоит смотреть в первую очередь.

❌ Блок «Нет в справочнике»

Если вставил пачку кодов, и каких-то нет в базе — под результатами появится список ненайденных Event ID. Полезно: сразу видно, что нужно проанализировать самостоятельно.

Вставил 4624 4625 9009 5555 → внизу: ❌ Нет в справочнике (2): 9009, 5555

Галочка «по заголовку»

По умолчанию поиск идёт по всем полям (описание, IoC, SPL…). Включи «по заголовку», чтобы искать только по заголовку события — меньше лишних результатов, когда знаешь название, а не номер.

🗂️ Фильтры и вкладки

Источник
  • Windows · Sysmon · Linux
  • ⚔️ Атака / 🛡️ Защита — инструменты
  • ❌ Error-коды · 🔑 Logon Types
  • 🧩 Цепочки атак (storylines)
Ещё фильтры
  • Категория — Logon, RDP, AD, ⭐ Популярные… (зависит от источника)
  • Важность — можно выбрать несколько сразу (Крит. + Высокая)
  • MITRE — по конкретной технике (T1078…)
Фильтры складываются с поиском. Кнопка «✕ сбросить фильтры» вернёт всё как было.

🔗 Связи между событиями

  • В блоке «Корреляции» номера событий кликабельны — клик по 4740 перекинет на это событие.
  • Теги MITRE в карточке кликабельны — покажут все события с этой техникой.
  • Блок «Участвует в цепочках атак» — переход к сценарию, частью которого является событие.

🧩 Цепочки атак

Вкладка «🧩 Цепочки атак» — 24 типовых сценария (Brute-force, Kerberoasting, DCSync, Ransomware, PetitPotam…). В каждой карточке:

  • 💡 Что за атака? — свернётся/развернётся, объяснение простым языком.
  • Пошаговый разбор — шаг · Event ID · фаза · описание.
  • 🔎 показать все события цепочки — откроет все события разом.

⚙️ Удобства

  • 📋 Копировать SPL — кнопка на блоке SPL-запроса (вставляй прямо в Splunk).
  • ☑️ развернуть все — раскрыть сразу все карточки (удобно листать/сравнивать).
  • 🔗 ссылка — копирует URL с текущими фильтрами. Кинул коллеге — у него откроется та же выборка.
  • 📱 Офлайн — страницу можно установить как приложение (кнопка «Установить» в браузере); работает без интернета.
