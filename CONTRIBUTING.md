# 🤝 Как дополнять и исправлять справочник

Спасибо, что помогаешь! Здесь всё максимально просто — **git знать не нужно**, всё правится прямо в браузере на GitHub. Робот проверит твои изменения на ошибки, а после публикации сайт обновится сам.

## 🧭 Главное правило

Мы правим только файлы в папке [`json/`](json/). Всё остальное (`data.js`, страница поиска) собирается автоматически — **трогать их не надо**.

| Что правим | Файл |
|:--|:--|
| Событие Windows (добавить/исправить Event ID) | `json/WinEvents_ALL.json` |
| Инструмент атакующих | `json/Инструменты_атакующих.json` |
| Инструмент защитников | `json/Инструменты_защитников.json` |
| События Linux | `json/LINUX.json` |
| Sysmon | `json/8Sysmon.json` |

---

## ✏️ Как внести правку (в браузере, без установки)

1. Открой нужный файл на GitHub и нажми **карандаш ✏️** (Edit this file).
2. GitHub сам сделает копию (форк) — просто согласись.
3. Внеси изменение (добавь или поправь запись — примеры ниже).
4. Внизу нажми **Commit changes** → **Create a new branch and start a pull request**.
5. Нажми **Create pull request**. Готово!

Дальше:
- 🤖 **Робот проверит** твой JSON (30–60 сек). Если что-то не так — покажет **красный крестик** и подскажет, что именно (например: «поле Важность = "Крит.", допустимо: Критическая/Высокая/…»).
- ✅ Если всё хорошо — **зелёная галочка**, и мейнтейнер примет изменение.
- 🚀 После принятия **сайт обновится автоматически** за ~минуту.

> Сломать ничего нельзя: если в записи ошибка, робот не даст её опубликовать.

## 📋 Шаблоны записей (копируй и меняй)

### Добавить событие Windows (`WinEvents_ALL.json`)

Добавь объект в конец массива (перед последней `]`), не забудь запятую после предыдущего:

```json
{
  "Категория": "Аутентификация",
  "Event ID": "4625",
  "Название события": "Неудачная попытка входа",
  "Лог": "Security",
  "Важность": "Высокая",
  "Частота": "Средняя",
  "Описание": "Кратко: что произошло",
  "Подробности": "Что содержит запись события",
  "Индикаторы атаки": "• признак 1\n• признак 2",
  "False Positive (легитимные срабатывания)": "• когда это норма",
  "MITRE TTP": "T1110, T1110.001",
  "SPL-шаблон": "index=wineventlog EventCode=4625 | stats count by ...",
  "Корреляции": "4740 (lockout), 4624 (успех после серии)",
  "Документация для данного EventCode": "https://learn.microsoft.com/..."
}
```

Обязательные поля: **`Event ID`** и **`Название события`**.
`Важность` — только: `Критическая` · `Высокая` · `Средняя` · `Низкая` · `Информационное`.
`Частота` — только: `Очень высокая` · `Высокая` · `Средняя` · `Низкая` · `Очень низкая`.

### Добавить инструмент атакующих (`Инструменты_атакующих.json`)

```json
{
  "id": "75",
  "category": "Credential Access",
  "name": "Название инструмента",
  "type": "Standalone / C2 / …",
  "platform": "Windows / Linux",
  "description": "Что делает, 1–2 предложения",
  "process_names": "имена процессов/файлов — важно для детекта",
  "mitre_ttp": "T1003, T1555",
  "event_ids": "Sysmon 1, 10; 4688 …",
  "reference": "github.com/…  (можно без https://)",
  "patterns": "паттерны в логах"
}
```

Обязательные: **`name`** и **`category`**.

### Добавить инструмент защитников (`Инструменты_защитников.json`)

```json
{
  "id": "191",
  "category": "EDR / XDR",
  "name": "Название",
  "type": "Commercial / Open Source",
  "platform": "Windows/Linux/macOS",
  "description": "Что это и что умеет",
  "process_names": "процессы/агенты (если есть)",
  "d3fend": "D3-PA, D3-NTA",
  "capabilities": "какую телеметрию собирает / возможности",
  "reference": "example.ru"
}
```

Обязательные: **`name`** и **`category`**.

## ✅ Частые ошибки (их ловит робот)

- **Лишняя запятая** перед `]` или `}` — самая частая. Уберите её.
- **Опечатка в «Важность»** — пишите ровно как в списке выше.
- **Пропущено `name` / `Event ID`** — без них запись не отобразится.
- **Кавычки** — только прямые `"`, каждое поле в кавычках.

## 🛠️ Для продвинутых (необязательно)

Проверить данные локально до PR:
```bash
pip install jsonschema
python3 schemas/validate.py       # проверка данных
python3 build.py                  # пересборка data.js (обычно делает робот)
```

## 🙋 Не хочешь возиться с JSON?

Просто открой **Issue** (вкладка Issues → New issue → «Предложить/исправить») и опиши, что добавить или поправить — оформим сами.
